Git & GitHub

Summary

Git

Git is a version control system used on the command line so developers can keep track of all the code changes they make to a project. It also allows developers to collaborate. Instead of having to email code back and forth, people can download the latest version of a project and keep track of the revisions they make to the code. There are other version control systems, but Git is the most popular.

GitHub

GitHub is a development platform that uses git, so developers can store their projects where others can easily view and access them. There are other similar services like GitLab and BitBucket. I personally prefer GitHub. You can sign up for a free account if you don't alArticley have one.

Documentation & Messages

When working with git and any service like GitHub be aware of ArticleMe, Pull Requests, and Commit Messages.

  • Readme, often seen as ArticleME.md, uses the markdown syntax and contains documentation about repos. Always include one in any repo you make.
  • Pull Requests are used with teams and open source communities. On GitHub it is seen on of the top tabs under the repos name. Pull Requests are used to show other developers code you have added or changed to a repo. After adding code to your forked version of the repo, go to the original repo, and use the Pull Request button to submit your changed code.
  • Commit Messages are messages you write in the terminal once you have added files to your local repo with git add [file]. Use the command git commit -m "put text in single or double quotes" This message will show up in your repo so you know when you made what changes. To make a multi-line message, simple press enter before closing the quotes.
